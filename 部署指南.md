# 云途智囊 · 跨电脑部署指南

## 先理解：部署什么东西

整个云途智囊系统由两部分文件组成：

### 第一部分：项目文件（放在项目目录里，跟着 git 走）

这些文件在 `c:\Users\赖函\Desktop\yuntushijie\` 下面，你克隆仓库就会自动获得：

```
你的项目目录/
├── CLAUDE.md                           ← 告诉 Claude Code 代理怎么设置
├── .claude/                            ← Claude Code 的配置目录
│   ├── settings.json                   ← 项目权限规则、技能注册
│   ├── settings.local.json             ← 你本机专属的权限（不共享给别的电脑）
│   ├── skills/                         ← 6个 Agent 的技能定义文件
│   │   ├── orchestrator.md             ← 总调度师
│   │   ├── marketing-agent.md          ← 营销运营官
│   │   ├── route-planner.md            ← 线路规划师
│   │   ├── image-qa.md                 ← 影像智检师
│   │   ├── activity-scheduler.md       ← 活动排期策略师
│   │   └── poster-designer.md          ← 海报设计师
│   │   └── .claude-plugin/             ← 技能注册配置（让 Claude Code 识别这些技能）
│   └── knowledge-base/                 ← 知识库（数据文件）
│       ├── route-library.json          ← 41条路线数据
│       ├── leader-roster.json          ← 领队信息
│       ├── brand-guidelines.json       ← 品牌规范
│       ├── material-library.json       ← 素材库
│       └── activity-schedule.json      ← 排期数据
├── 路线库.html                         ← 可视化路线库页面
├── 本周排期_5月25-31日.md               ← 运营文档
├── 沙河坝_社群文案_5月28日.md           ← 文案文档
├── 嘉措琼嘉_613-614_预热排期_Blogger合作Brief.md
└── ...                                 ← 其他文件
```

### 第二部分：用户级文件（放在你的用户目录里，不跟 git 走）

这些文件在 `C:\Users\赖函\.claude\`（也就是 `~/.claude/`）下面，**每台电脑都需要单独配置**：

```
~/.claude/skills/                       ← Claude Code 启动时自动扫描这个目录
    ├── orchestrator.md                 ← 和项目里的同名文件内容一样
    ├── marketing-agent.md
    ├── route-planner.md
    ├── image-qa.md
    ├── activity-scheduler.md
    └── poster-designer.md
```

> **为什么要存两份？**  
> 项目里的 `.claude/skills/` 用来在 git 上共享、版本管理。  
> 用户目录里的 `~/.claude/skills/` 是 Claude Code 启动时必定加载的位置——这是软件本身的机制。  
> 两处内容完全一样，作用互补：项目那套跟着代码走（团队共享），用户那套确保启动时一定被发现。

## 第一步：把现有项目上传到 Git 仓库

> **前置条件：** 你需要一个 Git 远程仓库。如果你还没有，先去 [GitHub](https://github.com) 或 [Gitee](https://gitee.com) 创建一个空仓库（不要勾选"初始化 README"），记下仓库地址，类似 `https://github.com/你的用户名/yuntushijie.git`

### 1.1 打开终端

在项目目录 `c:\Users\赖函\Desktop\yuntushijie` 里打开终端（或 VS Code 的终端），执行以下每条命令。

### 1.2 创建 .gitignore 文件

这个文件告诉 git **哪些文件不要上传**。`settings.local.json` 里有你本机专属的权限配置（包含你的用户名路径），不应该上传到别的电脑。

```bash
# 进入项目目录
cd /c/Users/赖函/Desktop/yuntushijie

# 创建 .gitignore，写入要忽略的文件名
cat >> .gitignore << 'EOF'
# 本地配置文件（每台电脑不同，不要上传）
.claude/settings.local.json

# macOS 系统文件
.DS_Store

# Windows 缩略图缓存
Thumbs.db
EOF
```

> **解释：** `cat >> .gitignore << 'EOF' ... EOF` 的意思是"把下面这几行文字写入 .gitignore 文件"。`#` 开头的是注释，给人看的，不影响执行。

### 1.3 提交所有文件

```bash
# 把当前目录下所有文件加入 git 的"待提交列表"
git add .

# 确认一下哪些文件会被提交（检查有没有不该提交的）
git status

# 如果上面看到有不该提交的文件（比如 .env、密码文件），用这个命令取消：
# git rm --cached 文件名

# 创建第一个提交
git commit -m "初始化云途智囊：6个Agent + 41条路线库 + 知识库 + 本周排期"
```

> **解释：**  
> `git add .` — 把当前目录所有文件标记为"准备提交"  
> `git commit -m "消息"` — 真正保存一次版本快照，消息描述这次改了什么

### 1.4 推送到远程仓库

```bash
# 关联远程仓库（把下面这行里的地址换成你自己的仓库地址）
git remote add origin https://github.com/你的用户名/yuntushijie.git

# 推送到远程仓库
git push -u origin main
```

> **解释：**  
> `git remote add origin 地址` — 告诉 git "远程仓库在哪里"，origin 是给远程仓库起的别名  
> `git push -u origin main` — 把本地代码推上去，`-u` 让以后直接 `git push` 就行

## 第二步：在新电脑上部署

拿到新电脑后，按顺序执行下面三个步骤。

### 步骤 1：克隆项目到新电脑

```bash
# 进入你想放项目的目录（桌面为例）
cd ~/Desktop

# 从远程仓库下载整个项目
git clone https://github.com/你的用户名/yuntushijie.git

# 进入项目目录
cd yuntushijie
```

> **解释：** `git clone` 会把远程仓库的所有文件下载到本地，自动创建一个同名文件夹。

此时新电脑上已经有了第一部分的所有文件（项目文件），但**技能还不能用**——因为第二部分（用户级文件）还没装。

### 步骤 2：安装技能到用户目录（最关键的一步）

Claude Code 启动时会自动扫描 `~/.claude/skills/` 目录，把里面的 `.md` 文件加载为可用技能。我们要把项目里的 6 个 Agent 技能文件复制过去。

```bash
# 先创建用户级技能目录（如果还不存在）
mkdir -p ~/.claude/skills

# 逐个复制 6 个技能文件
# （从项目目录 .claude/skills/ 复制到用户目录 ~/.claude/skills/）
cp .claude/skills/orchestrator.md     ~/.claude/skills/
cp .claude/skills/marketing-agent.md  ~/.claude/skills/
cp .claude/skills/route-planner.md    ~/.claude/skills/
cp .claude/skills/image-qa.md         ~/.claude/skills/
cp .claude/skills/activity-scheduler.md ~/.claude/skills/
cp .claude/skills/poster-designer.md  ~/.claude/skills/
```

> **用一行命令搞定（可选）：**
> ```bash
> for f in orchestrator marketing-agent route-planner image-qa activity-scheduler poster-designer; do
>   cp ".claude/skills/${f}.md" ~/.claude/skills/
> done
> ```
> 
> **解释：** 这是一个 bash 循环，依次把每个文件名拼出来然后复制。和上面 6 行 cp 命令效果完全一样。

**验证复制成功：**

```bash
# 列出用户技能目录，应该看到 6 个文件
ls ~/.claude/skills/
```

输出应该类似：
```
activity-scheduler.md  image-qa.md       orchestrator.md
marketing-agent.md     poster-designer.md  route-planner.md
```

### 步骤 3：配置网络代理（如果需要）

如果你的新电脑也需要用 Clash Verge 翻墙访问 Claude API，执行以下配置：

```bash
# 把代理环境变量写入 bash 启动配置
cat >> ~/.bashrc << 'EOF'

# === 云途智囊专用：Clash Verge 代理 ===
export HTTP_PROXY="http://127.0.0.1:7897"
export HTTPS_PROXY="http://127.0.0.1:7897"
EOF

# 让配置在当前终端立即生效
source ~/.bashrc
```

> **解释：**  
> `~/.bashrc` — bash shell 每次启动时自动执行的脚本，放环境变量最合适  
> `export HTTP_PROXY="..."` — 设置 HTTP 代理，让所有网络请求走 Clash Verge  
> `source ~/.bashrc` — 重新加载配置文件，不用关终端就能生效  
> **注意：** 新电脑上需要先安装 Clash Verge 并启动，代理地址才是 `127.0.0.1:7897`。如果用的是其他代理软件，端口号可能不同。

## 第三步：验证部署成功

在新电脑上打开终端，进入项目目录，启动 Claude Code：

```bash
cd ~/Desktop/yuntushijie
claude
```

进入 Claude Code 后，依次测试三个技能：

| 输入 | 预期效果 |
|------|---------|
| `/orchestrator` | 看到 🟠 总调度师回复，列出 5 个 Agent 目录 |
| `/activity-scheduler 帮我排6月活动` | 看到 🔵 日历 Agent 按五步流程输出排期 |
| `/marketing-agent 帮我写一条赤水瀑布群的朋友圈文案` | 看到 ❤️ 营销官输出文案 |

如果上面三个都能正常回复，说明部署成功。

> **常见问题：**
> - 如果 `/orchestrator` 提示 "Unknown skill"：检查步骤 2 是否执行成功（`ls ~/.claude/skills/` 看有没有文件）
> - 如果 Claude Code 网络报错：检查代理是否启动，`echo $HTTP_PROXY` 看环境变量是否设置

## 第四步：日常使用——两台电脑之间同步数据

假设你有两台电脑：**电脑A**（现在的）和**电脑B**（新部署的）。日常使用中，你可能会在任意一台电脑上修改知识库、添加新路线、更新排期等。

### 每次开始工作前（拉取最新数据）

```bash
cd ~/Desktop/yuntushijie
git pull
```

> **解释：** `git pull` — 从远程仓库下载别人（或另一台电脑上的你）更新过的文件，合并到本地

### 每次工作结束后（推送你的修改）

```bash
cd ~/Desktop/yuntushijie
git add .
git commit -m "简单描述你改了什么，比如：更新6月排期、新增XX路线"
git push
```

> **解释：**  
> `git add .` → `git commit` → `git push` 三步走：标记修改 → 保存快照 → 传到远程

### 什么文件改了需要推送

| 你做了什么 | 需要 push 吗 |
|-----------|-------------|
| 用 AI 更新了路线库（route-library.json） | ✅ 是 |
| 修改了领队信息（leader-roster.json） | ✅ 是 |
| 生成了新的活动排期 | ✅ 是 |
| 添加了新的 Agent 技能 | ✅ 是，而且新电脑也要重新执行步骤 2 |
| 修改了品牌规范或素材库 | ✅ 是 |
| 只是普通的聊天对话 | ❌ 不用 |

> **重要提醒：**
> - 知识库数据（`.claude/knowledge-base/` 下的所有 JSON 文件）改了之后一定要 push，否则另一台电脑看不到
> - 如果你新增了 Agent（比如第 7 个技能文件），除了 push 项目文件，还需要在新电脑上**重新执行第二步的复制命令**把新文件拷到 `~/.claude/skills/`
